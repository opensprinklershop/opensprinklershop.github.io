Référence API du firmware 2.2.1(4) [1 novembre 2025]

Cette référence traduite décrit l'API du firmware OpenSprinkler. Les chemins d'API, paramètres, noms de variables et champs JSON restent inchangés afin que les exemples fonctionnent directement avec le firmware.

0. Vue d'ensemble

  • Authentification (pw) : tous les appels API nécessitent le mot de passe de l'appareil via pw, sous forme de hash MD5 en minuscules. L'application et l'interface Web le calculent automatiquement.
  • Adresse de l'appareil : os-ip désigne l'adresse IP ou le nom d'hôte du contrôleur.
  • Paramètres : l'ordre des paramètres est libre. Les paramètres omis conservent leur valeur actuelle.
  • Réponses : les retours sont en JSON, par exemple {"fwv":221} ou {"result":1}.
Code Signification
1 Succès
2 Non autorisé, mot de passe manquant ou incorrect
3 Nouveau mot de passe et confirmation différents
16 Données requises manquantes
17 Valeur hors plage
18 Erreur de format de données
19 Format de code RF invalide
32 Page/fichier introuvable
48 Opération non autorisée
64 Échec de l'envoi, par exemple mise à jour OTA

1. Changements importants

  • OpenSprinkler v3.4 DC : nouvelles valeurs USB-PD tpdv (tension cible dans /jo) et apdv (tension active dans /jc).
  • Option de file d'attente : /cm, /cr et /mp acceptent qo pour ajouter, insérer en tête ou remplacer les exécutions existantes.
  • Les passages surlignés en vert indiquent les changements récents.

2. Lire les variables du contrôleur /jc

Utilisation : http://os-ip/jc?pw=xxx

Variable Signification
devt Heure locale de l'appareil en secondes Unix
nbrd Nombre de groupes de 8 zones, contrôleur principal inclus
en Fonctionnement activé (1) ou désactivé (0)
sn1, sn2 État des capteurs 1/2
rd, rdst Délai pluie actif et heure de fin
sunrise, sunset Lever/coucher du soleil en minutes depuis minuit
lwc, lswc Dernière requête/réponse météo
lrbtc Cause du dernier redémarrage
lrun Dernière exécution [station, programme, durée, fin]
RSSI, mac, loc Signal WiFi, adresse MAC, position
jsp, wsp, wto, wtdata, wterr UI, météo, options météo et erreurs
mqtt, email, ifkey, otc, otcs Intégrations et état de connexion
curr, ocs Courant total et état de surintensité
sbits, ps, nq États de stations, programmes et file d'attente
flwrt, flcrt, flcto Fenêtre de débit, impulsions courantes et cumulées
pq, pt Pause active et temps restant
gpio Broches GPIO disponibles
wtrestr, wls Restrictions météo et niveaux d'arrosage multi-jours
apdv Tension USB-PD active en volts, DC v3.4 seulement.

3. Modifier les variables du contrôleur /cv

Utilisation : http://os-ip/cv?pw=xxx&rsn=x&rrsn=x&rbt=x&en=x&rd=x&re=x&update=x&ap=x&rocs=x

rsn réinitialise toutes les stations, rrsn seulement les stations actives, rbt redémarre, en active/désactive, rd règle le délai pluie, re active le mode extension distante, ap relance la configuration WiFi, update déclenche une mise à jour OSPi/Linux et rocs efface l'état de surintensité.

4. Lire les options /jo

Utilisation : http://os-ip/jo?pw=xxx

Les options RO sont en lecture seule. Les principaux groupes sont version (fwv, fwm), réseau (dhcp, ip*, gw*, dns*, subn*), NTP, port HTTP, matériel (hwv, hwt), extensions, délais, stations maîtres, capteurs, météo, journalisation, débit, notifications, seuils de courant et tpdv pour la tension USB-PD cible.

5. Modifier les options /co

Utilisation : http://os-ip/co?pw=xxx&[opn]=[opv]&loc=x&wto=x&ifkey=x&ttt=x&otc=x&dname=x&mqtt=x&email=x

[opn] est le nom d'option de /jo, [opv] sa nouvelle valeur. Plusieurs options peuvent être modifiées en un seul appel. Les champs JSON sont URL-encodés sans les accolades extérieures.

6. Définir le mot de passe /sp

Utilisation : http://os-ip/sp?pw=xxx&npw=xxx&cpw=xxx

npw et cpw sont les hashes MD5 du nouveau mot de passe.

7. Lire noms et attributs de stations /jn

Utilisation : http://os-ip/jn?pw=xxx

Retourne snames, maxlen, stn_grp, les bitfields de maîtres, pluie/capteurs ignorés, stations désactivées et stations spéciales.

8. Lire les données de stations spéciales /je

Utilisation : http://os-ip/je?pw=xxx

Format : {"0":{"st":xxx,"sd":"xxx"}, ...}. Types : RF, Remote-IP, GPIO, HTTP, HTTPS et Remote OTC.

9. Modifier noms et attributs de stations /cs

Utilisation : http://os-ip/cs?pw=xxx&s?=x&m?=x&i?=x&j?=x&k?=x&n?=x&d?=x&p?=x&g?=x&sid=xx&st=xx&sd=xx

Permet de modifier noms, maîtres, bits d'ignorance, stations désactivées, groupes séquentiels et données de stations spéciales.

10. Lire l'état des stations /js

Utilisation : http://os-ip/js?pw=xxx

Retourne sn comme tableau 1/0 et nstations.

11. Démarrer une station manuellement /cm

Utilisation : http://os-ip/cm?pw=xxx&sid=xx&en=x&t=xxx&ssta=x&qo=x

sid est l'index de station, en ouvre/ferme, t est la durée, qo règle la file et ssta décale les stations restantes du même groupe.

12. Démarrer un programme manuellement /mp

Utilisation : http://os-ip/mp?pw=xxx&pid=xx&uwt=x&qo=x

qo=0 ajoute, qo=1 insère en tête, qo=2 remplace.

13. Lire les programmes /jp

Utilisation : http://os-ip/jp?pw=xxx

Retourne les limites et pd. Un programme suit [flag, days0, days1, [start0..start3], [dur...], name, [endr, from, to]].

14. Modifier les programmes /cp

Utilisation : http://os-ip/cp?pw=xxx&pid=xx&en=x&uwt=x&name=xxx&v=[...]&from=xxx&to=xxx

pid=-1 ajoute un programme. en et uwt modifient seulement les bits correspondants.

15. Supprimer des programmes /dp

Utilisation : http://os-ip/dp?pw=xxx&pid=x. pid=-1 supprime tous les programmes.

16. Déplacer un programme vers le haut /up

Utilisation : http://os-ip/up?pw=xxx&pid=x.

17. Programme ponctuel /cr

Utilisation : http://os-ip/cr?pw=xxx&t=[x,x,...]&cnt=xxx&int=xxx&uwt=xxx&anno=xxx&qo=x

t contient les durées par station, cnt/int les répétitions, uwt l'ajustement météo, anno l'annotation, qo la file.

18. Lire le journal /jl

Utilisation : http://os-ip/jl?pw=xxx&start=xxx&end=xxx&type=x ou http://os-ip/jl?pw=xxx&hist=n&type=x.

Entrées : [pid,sid,dur,end] ou [pid,sid,dur,end,flow]. Événements spéciaux : s1, s2, rd, fl, wl.

19. Supprimer le journal /dl

Utilisation : http://os-ip/dl?pw=xxx&day=n. day=all supprime tout.

20. Modifier les URL de scripts /cu

Utilisation : http://os-ip/cu?pw=xxx&jsp=xxx&wsp=xxx.

21. Tout lire /ja

Utilisation : http://os-ip/ja?pw=xxx. Combine /jc, /jo, /jn, /js et /jp.

22. Mettre la file en pause /pq

Utilisation : http://os-ip/pq?pw=xxx&dur=x&repl=x.

23. Diagnostic /db

Utilisation : http://os-ip/db. Aucun mot de passe requis.

24. Causes de redémarrage

Codes principaux : 0 inconnu, 1 réinitialisation usine, 2 bouton, 3 mode AP, 4 API/timer, 5 redémarrage API, 6 AP vers client, 7 mise à jour firmware, 8 météo échouée plus de 24 h, 9 réseau échoué trop souvent, 10 NTP, 99 mise sous tension.

25. Codes d'erreur météo

0 succès, -1 requête non reçue, -2 connexion impossible au serveur météo, -3 délai dépassé, -4 réponse vide. Les codes positifs viennent du service météo.
